## 风险 / 教训

1. **vision 单点故障** — 关键决策节点(像今天"持仓下方为何空")被迫改用代码 + 文字诊断,丢失视觉反馈。**对策**:把 vision_analyze 留作备用,关键改动后立刻用 vision 验一遍截图。
2. **多次 uncommitted 改动堆积** — `.gitignore` / 14 品种 / 面板 / 持仓 4 块改动没分开 commit,中间悟空"先不要急"时差点丢。**对策**:每个 PRG(单一目的)立刻 commit,不囤。
3. **误插 `</script>` 事故** — patch 时把 `</script>` 写到中段,把 doBuy/toggleMode/toggleOnlyHeld 踢出 script 块,页面静默崩。**对策**:patch 后 `grep -c "</script>"` + `grep -n "^function "` 必查。
